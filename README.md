# Olist Ecommerce Data Analysis

## Project Approach

This project follows a stakeholder-first analytical approach, where business questions are defined before analysis begins.

### Step 1: Stakeholder Simulation using LLM

Instead of manually defining the problem, a Large Language Model was used to simulate a Head of Growth stakeholder.The dataset context was provided, and the model was prompted to:

  - Define business challenges
  - Frame a problem statement
  - Generate key analytical questions

This helped replicate a real-world scenario where analysts receive ambiguous or high-level business requirements.

### Step 2: Validation & Refinement

The generated brief was reviewed and refined to ensure (BusinessStakeholderBreif) :

Questions were measurable using available data
Scope aligned with dataset limitations
Outputs would lead to actionable insights

### Step 3: Data Analysis

All analysis in this project is directly mapped to the stakeholder questions, ensuring:

#### Business relevance
- Clear decision-making outcomes
- Minimal exploratory noise
  
#### Why this approach stands out
- Mimics real-world analyst–stakeholder interaction
- Demonstrates ability to translate vague business needs into data problems
- Uses AI as a thinking partner, not just a tool
